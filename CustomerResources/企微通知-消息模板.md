# 企微通知消息模板

## 概述

系统通过企业微信应用消息通知相关人员线索分配状态，共五种消息类型：

1. **销售领取通知** - 线索分配给销售，等待确认领取
2. **人工池预警通知** - 线索进入人工池，通知管理员处理
3. **指定分配通知** - 管理员指定分配后通知销售跟进
4. **重复线索提醒** - 推广员录入重复线索时，通知原销售跟进
5. **优先分配通知** - 销售获得优先分配资格后，新线索优先分配时通知

---

### 领取链接处理逻辑

| 场景 | 处理结果 | 提示文案 |
|-----|---------|---------|
| 未超时点击 | 领取成功 | "✅ 领取成功！线索已归属您，请及时联系客户。" |
| 已超时点击 | 领取失败 | "⏰ 已超时！该线索已流转给其他销售。" |

### 链接安全机制
- 链接含唯一 Token，单次有效
- Token 有效期按渠道配置（X 分钟）
- 领取成功后 Token 立即失效
- 超时后 Token 自动失效

// Token 结构（JWT）

### 6.3 领取验证流程

```
销售点击链接
    │
    ▼
验证 Token 有效性
    │
    ├── Token 已过期 ────→ 提示"已超时"
    │
    └── 验证通过
            │
            ▼
    检查线索当前状态
            │
            ├── 状态不是"待确认" ──→ 提示"已超时"
            │
            └── 状态正常
                    │
                    ▼
            更新线索归属
            记录分配历史
            发送成功通知
```

### 6.4 失败重试机制

| 场景 | 重试策略 | 降级处理 |
|-----|---------|---------|
| 企微接口调用失败 | 重试3次，间隔10秒 | 记录日志，标记为"通知失败"，继续超时逻辑 |
| Token 生成失败 | 重试3次 | 告警管理员，线索进入人工池 |
| 回调处理异常 | 重试3次 | 记录异常，人工介入处理 |

## 七、通知配置

### 5.1 渠道通知配置

在渠道配置页面设置：

```
┌─────────────────────────────────────────┐
│ 兜底通知配置（必填）                      │
├─────────────────────────────────────────┤
│                                         │
│  当线索进入人工池时，通知以下管理员：      │
│                                         │
│  [销售主管小王 ▼] [运营小李 ▼] [+添加]   │
│                                         │
│  通知方式：                              │
│  ☑ 企微应用消息                          │
│  ☐ 短信通知（备用）                       │
│                                         │
│  通知说明：                              │
│  ● 线索进入人工池时即时通知所有配置的管理员 │
│                                         │
└─────────────────────────────────────────┘
```

### 5.2 通知模板配置

支持在系统后台配置通知文案：

| 配置项 | 默认值 | 说明 |
|-------|-------|------|
| 领取有效期 | 按渠道配置 | 销售确认领取的超时时间 |
| 提醒间隔 | 不适用 | 人工池线索仅通知一次，不重复提醒 |
| 最大提醒次数 | 3次 | 避免过度打扰管理员 |
