## 一、产品概述

### 1.1 产品背景

市场部通过多个推广渠道获取客户线索，需要一套自动化的分配系统将线索及时、公平地分配给销售团队进行跟进。

### 1.2 产品目标

| 目标 | 衡量指标 |
|-----|---------|
| 自动化分配 | 自动分配率 > 90% |
| 快速响应 | 平均响应时间 < 3分钟 |
| 零线索丢失 | 线索丢失率 = 0 |

### 1.3 用户角色

| 角色 | 职责 |
|-----|------|
| 推广员 | 渠道配置、线索监控、人工分配、录入线索 |
| 销售人员 | 接收线索通知、确认领取、跟进客户 |
| 系统管理员 | 基础数据维护 |

## 二、核心概念

| 术语 | 定义 |
|-----|------|
| **渠道** | 推广员-平台的组合（如：张三-小红书），关联一个销售队列进行分配 |
| **待分配队列** | 某渠道内尚未开始分配的线索池 |
| **人工池** | 某渠道内所有销售都超时后进入的线索池，需人工处理 |
| **销售队列** | 独立管理的销售分组，一个队列可关联多个渠道 |
| **销售状态池** | 全局共享，记录每个销售当前是"空闲"还是"待确认" |
| **兜底管理员** | 渠道配置的管理人员，当线索进入人工池时接收通知 |

### 2.1 渠道与销售队列关系

```
销售队列（独立管理）          渠道（推广员+平台）
├── 华东销售一组 ───────→  ├── 张三-小红书
│   ├── 销售A              ├── 张三-抖音
│   ├── 销售B              └── 李四-小红书
│   └── 销售C              
│                          
├── 华东销售二组 ───────→  ├── 王五-百度
│   ├── 销售D              
│   └── 销售E              
│                          
└── VIP客户组 ─────────→   └── 钱七-线下展会
    ├── 销售A
    └── 销售D
```

**核心特性**：
- 销售队列独立管理，可复用于多个渠道
- 一个渠道只能关联一个销售队列
- 一个销售队列可关联多个渠道
- 销售状态全局共享（某销售在渠道A待确认时，渠道B也会跳过他）

## 三、功能模块

### 3.1 渠道管理

| 功能 | 说明 |
|-----|------|
| 渠道列表 | 展示所有渠道，支持增删改查和状态管理 |
| 渠道配置 | 选择销售队列、配置超时时间、兜底管理员 |
| 录入线索 | 手动录入线索到指定渠道，支持查重验证 |

**渠道配置项**：

| 配置项 | 说明 | 必填 |
|-------|------|------|
| 推广员 | 选择已维护的推广员 | 是 |
| 平台 | 选择已维护的平台 | 是 |
| 销售队列 | 从预设队列中选择 | 是 |
| 超时时间 | 每个渠道独立配置（X分钟） | 是 |
| 兜底管理员 | 1~N人，线索进人工池时通知所有配置的管理员 | 是（至少1人） |

**线索录入查重**：
- 基于客户手机号或微信号进行查重
- 发现重复时阻止录入，并通知原销售及时跟进


**销售队列管理功能**：
- 新建/编辑销售队列
- 拖拽排序销售顺序
- 查看队列关联的渠道

### 3.2 线索管理

| 功能 | 说明 |
|-----|------|
| 待分配队列 | 各渠道等待自动分配的线索 |
| 流转中监控 | 监控正在等待销售确认的线索，含倒计时 |
| 人工池 | 管理超时未分配的线索，支持手动分配、指定分配 |
| 分配记录 | 查询所有线索的完整流转历史 |

**人工池-指定分配**：
- 支持多选后批量指定分配
- 按钮旁提示："指定分配后，线索不会超时流转，请确保分配的销售小伙伴及时联系客户"

### 3.3 基础数据

| 功能 | 说明 |
|-----|------|
| 推广员管理 | 维护推广员列表 |
| 平台管理 | 维护平台字典 |
| 销售人员管理 | 同步企微成员，设置账号状态 |
| 销售队列管理 | 独立管理销售队列，支持复用于多个渠道 |

## 四、业务流程

### 4.1 自动分配流程

```
新线索进入渠道
    │
    ▼
检查是否有销售拥有优先分配资格
    │
    ├── 有优先资格且空闲 → 优先分配（临时置顶）
    │       │
    │       ▼
    │   发送优先分配通知
    │       │
    │       ▼
    │   销售确认领取 → 分配成功，消耗1个优先资格
    │       │
    │       ▼
    │   超时未确认 → 资格保留，流转给下一位
    │
    └── 无优先资格或销售不空闲
            │
            ▼
        按销售队列顺序尝试分配
            │
            ├── 销售空闲 → 分配，发送通知（X分钟有效期）
            │
            ├── 销售待确认 → 跳过，检查下一位
            │
            └── 销售异常 → 跳过，记录告警
                        │
                        ▼
                    确认领取 → 分配成功
                        │
                    超时 → 流转给下一位
                        │
                    全部超时 → 进入人工池 → 通知兜底管理员
```

### 4.2 关键业务规则

| 规则 | 说明 |
|-----|------|
| BR-01 | 渠道 = 推广员-平台，每个组合是一个独立渠道 |
| BR-02 | 销售队列独立管理，可关联多个渠道 |
| BR-03 | 销售状态（空闲/待确认）是全局共享的 |
| BR-04 | 每个渠道独立配置超时时间（X分钟） |
| BR-05 | 销售同时只能处理1条待确认线索 |
| BR-06 | 末位超时后进入人工池，不再自动流转 |
| BR-07 | 人工池线索手动分配后直接归属，无需确认 |
| BR-08 | 每个渠道必须配置至少1个兜底管理员 |
| BR-09 | 领取链接单次有效，超时即失效 |
| BR-10 | 线索录入时查重，重复则阻止录入并通知原销售 |
| BR-11 | 销售反馈线索无效后，获得优先分配资格（可累积） |
| BR-12 | 优先分配资格无有效期，直到成功领取线索后消耗 |
| BR-13 | 优先分配仍需销售确认领取，不是直接归属 |
| BR-14 | 优先分配超时后，资格保留，线索正常流转 |
| BR-15 | 管理员不能强制回收已分配但未确认的线索 |
| BR-16 | 调整销售顺序下条线索生效，当前流转中的线索保持原顺序 |
| BR-17 | 从渠道移除销售时，该渠道正在流转中的线索继续完成分配 |
| BR-18 | 账号异常的销售自动跳过，并告警管理员 |
| BR-19 | 全渠道销售都忙碌时，新线索直接进入该渠道人工池（不等待） |
| BR-20 | 分配历史数据永久保留 |

## 五、企微通知

系统通过企微应用消息通知相关人员，共五种消息类型：

| 类型 | 触发场景 | 接收人 |
|-----|---------|-------|
| 销售领取通知 | 线索自动分配给销售 | 被分配销售 |
| 人工池预警通知 | 线索进入人工池 | 兜底管理员 |
| 指定分配通知 | 管理员手动分配线索 | 被指定销售 |
| 重复线索提醒 | 推广员录入重复线索 | 原负责销售 |
| 优先分配通知 | 销售获得优先分配资格后，新线索优先分配 | 获得优先资格的销售 |

**详细消息模板见**：《企微通知消息模板.md》

### 领取链接处理

| 场景 | 结果 | 提示文案 |
|-----|------|---------|
| 未超时点击 | 领取成功 | "✅ 领取成功！线索已归属您，请及时联系客户。" |
| 已超时点击 | 领取失败 | "⏰ 已超时！该线索已流转给其他销售。" |
